Sintaxis general

Cada pregunta inicia con N. donde N es el número. Los campos opcionales van entre corchetes al inicio:

  • [puntos:N] — puntuación del reactivo (opcional, default = 1)
  • [tipo:T] — tipo de pregunta (opcional, default = multiple_choice)

Marcas especiales

  • * antes de la letra → marca la respuesta correcta
  • + al inicio de línea → retroalimentación para respuesta correcta
  • - al inicio de línea (con espacio) → retroalimentación para respuesta incorrecta
  • = al inicio de línea → respuesta correcta (short_answer, numeric) o ítem ordenado (ordering)
  • | separa término y definición en matching

Ejemplo 1: Opción múltiple (default)

1. [puntos:5] ¿Qué necesita un municipio para que sus datos generen valor?
*A) Un proceso analítico que convierta los datos en conocimiento útil.
B) Más sensores para ampliar la cobertura de captura.
C) Un sistema de almacenamiento mayor para el histórico.
D) Una red de transmisión más veloz.
+ ¡Correcto! Los datos requieren un proceso analítico para ser útiles.
- Incorrecto. Ampliar captura o almacenamiento no resuelve el problema.

Ejemplo 2: Respuesta múltiple

2. [puntos:4] [tipo:multiple_answer] ¿Cuáles son ejemplos de IA estrecha?
*A) Asistentes de voz como Alexa
*B) Sistemas de recomendación de Netflix
C) Un robot que aprende cualquier tarea nueva sin entrenamiento
*D) Chatbots como ChatGPT
+ Correcto: todas excepto el robot que aprende cualquier tarea sola.
- Recuerda: la IA actual es estrecha (ANI).

Ejemplo 3: Verdadero / Falso

3. [puntos:2] [tipo:true_false] La IA general (AGI) existe hoy comercialmente.
*A) Verdadero
B) Falso
+ ¡Atención! La afirmación dice que AGI existe, lo cual es FALSO.
- La IA actual es estrecha (ANI), no general.

Ejemplo 4: Ordenamiento

4. [puntos:3] [tipo:ordering] Ordena las etapas del proceso analítico:
=) Captura de datos
=) Limpieza y transformación
=) Análisis estadístico
=) Toma de decisiones
+ Correcto: captura → limpieza → análisis → decisión.
- Revisa el orden del flujo analítico.

Ejemplo 5: Emparejamiento

5. [puntos:4] [tipo:matching] Relaciona cada concepto con su definición:
A) Dato | Hecho bruto sin procesar
B) Información | Datos organizados con contexto
C) Conocimiento | Información interpretada
D) Sabiduría | Conocimiento aplicado con juicio
+ Correcto: es la jerarquía DIKW.
- Cada nivel agrega valor sobre el anterior.

Ejemplo 6: Respuesta corta

6. [puntos:3] [tipo:short_answer] ¿Qué sigla identifica a la IA General?
= AGI
= Inteligencia Artificial General
+ Correcto: AGI (Artificial General Intelligence).
- La sigla es AGI.

Ejemplo 7: Numérica

7. [puntos:2] [tipo:numeric] ¿Cuántos bits tiene un byte?
= 8
+ Correcto: 1 byte = 8 bits.
- 1 byte = 8 bits.

Consejos

  • El [puntos:N] puede ser decimal: [puntos:2.5]
  • Si omites + o -, esa retroalimentación no se generará
  • Si omites [tipo:T], se asume multiple_choice
  • El texto de la pregunta puede continuar en varias líneas antes de las opciones
  • Para múltiples respuestas correctas en short_answer, usa varias líneas =

- [tipo:T] = multiple_choice | multiple_answer | true_false |
  ordering | matching | short_answer | numeric
  (si es opción múltiple simple, omite [tipo:])
- Marca con * la(s) respuesta(s) correcta(s)
- Para ordering: usa "=) item" en el orden correcto
- Para matching: usa "A) término | definición"
- Para short_answer/numeric: usa "= respuesta"
- + = retroalimentación respuesta correcta
- - = retroalimentación respuesta incorrecta

🛠 Solución de problemas

  • No se importan las preguntas: verifica que el ZIP tenga 2 archivos (imsmanifest.xml y res00001.dat)
  • Faltan puntos: revisa que [puntos:N] esté al inicio de cada pregunta
  • Falta retroalimentación: revisa que las líneas + y - estén presentes
  • Caracteres raros: el parser escapa automáticamente < > & "
